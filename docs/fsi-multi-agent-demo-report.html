Executive summary

The demo presents three visible FSI analyst scenarios in a Next.js portal. The portal streams each multi-agent run from a FastAPI backend, which invokes Azure AI Foundry prompt agents through the OpenAI-compatible responses API. Agents use Code Interpreter to produce real spreadsheet and presentation artifacts and emit traces to Application Insights.

Runtime sequence

  1. The user opens the public portal hosted by Azure Container Apps.
  2. The portal loads scenarios and toolbox metadata from GET /api/scenarios and GET /api/toolboxes.
  3. The user starts a scenario with POST /api/run; the API streams progress with Server-Sent Events.
  4. The API authenticates through the Container App's user-assigned managed identity and Azure RBAC.
  5. The API invokes Foundry prompt agents using responses.create and agent_reference.
  6. Specialist agents run in sequence and receive the user prompt, synthetic data context, and upstream outputs.
  7. Code Interpreter builds files; Web Search is available for grounding.
  8. Foundry returns container-file citations; the API downloads the files and exposes /api/artifacts/{artifact_id}.
  9. The portal renders artifact download links and the orchestrator's final synthesis.
  10. The API sends scenario.run and agent.turn spans to Application Insights.

Artifacts and observability

Artifacts

Generated workbooks and decks appear in the portal live stream. The API keeps a temp copy in the container and serves it through /api/artifacts/{artifact_id}. For durable links, persist generated files to Azure Storage Blob and return blob or SAS URLs.

Telemetry

The API uses azure-monitor-opentelemetry. Application Insights receives custom spans: scenario.run for the whole workflow and agent.turn for each specialist or orchestrator turn.

Path to live vendor data

The current demo is self-contained with synthetic data. To connect live FSI providers, store vendor credentials in Key Vault, add the provider as an MCP tool connection or toolbox tool in Foundry, attach it to the relevant specialist agents, and update the instructions to prefer live sources.
