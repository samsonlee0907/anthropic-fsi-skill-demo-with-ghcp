Executive summary

v3 moves the runtime from an API-hosted orchestrator-worker design to real Foundry hosted agents. The API is now a thin backend-for-frontend: it submits background Responses requests, polls for completion, streams status to the portal, and proxies generated artifacts from private Blob Storage.

The agent landscape is scenario-based. There are three hosted agents, not one agent per skill: fsi-equity-v3, fsi-ib-pitch-v3, and fsi-pe-lbo-v3.

Architecture

Container Apps

Portal: Next.js public UI.

API: FastAPI BFF with SSE, background polling, and artifact proxy endpoints.

Azure AI Foundry

Three hosted agents deployed into Foundry Agent Service with azd ai agent.

Each uses toolbox MCP for skills, a self-hosted SEC EDGAR remote MCP tool for public filings, plus native code_interpreter and web_search.

Storage and artifacts

Hosted-agent middleware uploads generated Office files to the private artifacts Blob container.

The browser downloads through /api/artifacts/{id}, not directly from Storage.

Identity and RBAC

The API uses a user-assigned managed identity.

Each hosted agent has its own instance identity with Foundry and Blob permissions.

Runtime flow

1
User opens the portal and selects one of the three scenarios.
2
The portal calls POST /api/run and receives Server-Sent Events.
3
The API submits a Foundry Responses request with background:true, store:true, and stream:false.
4
The selected hosted agent loads scenario skills through the toolbox MCP provider.
5
When a real public ticker is supplied, the hosted agent can use SEC EDGAR tools for company metadata, filing lists, 10-K/10-Q sections, and XBRL financials.
6
The hosted agent uses native Code Interpreter to create the workbook or deck.
7
Artifact egress middleware uploads generated files to private Blob Storage and appends an artifact sentinel to the final response.
8
The API parses the sentinel, downloads the blob privately, registers an artifact ID, strips the sentinel, and sends the clean result to the portal.

Operational commands

Health check

Invoke-RestMethod https://ca-api-fsi-demo-v3.politeocean-8e501b06.eastus2.azurecontainerapps.io/api/health

Redeploy hosted agents

cd C:\Users\samsonlee\GHCP\fsi-multiagent-demo\agents\hosted
Copy-Item .\fsi_hosted_agent_v3.py .\_azd\agent-src\fsi_hosted_agent_v3.py -Force
Copy-Item .\fsi_artifact_egress.py .\_azd\agent-src\fsi_artifact_egress.py -Force
cd .\_azd
$env:GH_TOKEN = gh auth token
$env:GITHUB_TOKEN = $env:GH_TOKEN
$env:AZD_AGENT_SKIP_ACR = "true"
azd deploy fsi-equity-v3
azd deploy fsi-ib-pitch-v3
azd deploy fsi-pe-lbo-v3

Rebuild API and portal

cd C:\Users\samsonlee\GHCP\fsi-multiagent-demo
az acr build --registry acrxzqm33pk --image fsi-api:v3 .\api
az acr build --registry acrxzqm33pk --image fsi-portal:v3 `
  --build-arg NEXT_PUBLIC_API_BASE_URL=https://ca-api-fsi-demo-v3.politeocean-8e501b06.eastus2.azurecontainerapps.io `
  .\portal
